# Lane-Level Vehicle Detection

基于 OpenCV 的高速公路车辆检测、车道检测与分车道车辆计数项目。当前版本面向单张高速路监控图像，完成车道边界提取、逆透视鸟瞰变换、车辆分割、车辆所属车道判断，并输出最终可视化结果和调试图。

## src 脚本说明

- `src/detection_main.py`：主程序入口，只负责组织整体流程：读取图像、调用车道检测、车辆检测、车道计数、结果绘制和调试图保存。
- `src/config.py`：项目路径和全局尺寸配置，包括输入图像路径、输出路径、debug 目录和 BEV 鸟瞰图尺寸。
- `src/geometry_utils.py`：几何工具函数，包括直线交点、消失点估计、IPM 梯形构建、一维寻峰、线段拟合和绘图线段延伸。
- `src/lane_detection.py`：车道检测与车道车辆计数逻辑，包括车道颜色 mask、Canny/Hough、IPM、车道边界合并，以及车辆到车道的归属判断。
- `src/vehicle_detection.py`：车辆检测逻辑，包括车辆 ROI、暗色/彩色/亮色/远处灰白车辆分支、形态学处理、凸包填充和连通域过滤。
- `src/visualization.py`：最终结果图和 debug 图绘制，包括车道线、最繁忙车道、车辆框、车道计数文字和 debug 拼图。

## debug 输出说明

- `debug/00_panel.jpg`：总览拼图，集中展示主要中间结果和最终输出。
- `debug/01_edges.jpg`：Canny 边缘检测结果，经过顶部区域裁剪，用于后续 Hough 线段检测。
- `debug/02_lane_color_mask.jpg`：白色车道线和黄色车道线的 HSV 颜色 mask。
- `debug/03_vehicle_mask.jpg`：车辆分割后的二值 mask，展示最终参与连通域过滤的车辆区域。
- `debug/04_ipm_trapezoid.jpg`：原图上的 IPM 源梯形和消失点可视化。
- `debug/05_bev_with_lanes.jpg`：鸟瞰图中的车道线和车辆投影位置。
- `debug/06_lane_histogram.jpg`：鸟瞰图车道线列直方图，用于展示车道边界寻峰结果。

部分 `_dbg_*` 图片是开发调试过程中保存的辅助图，不属于主流程必要输出。

## 整体 Pipeline

项目的整体流程可以概括为：

```text
输入图像
  -> HSV 颜色分解
  -> 白/黄车道线提取
  -> Canny + HoughLinesP 检测候选直线
  -> RANSAC 估计消失点
  -> 构建 IPM 源梯形并生成鸟瞰图
  -> BEV 列直方图寻峰得到内侧车道线
  -> 原图中补充检测外侧黄线和右路肩线
  -> 合并得到完整车道边界
  -> HSV + 纹理 + 形态学分割车辆
  -> 连通域过滤得到车辆 bbox
  -> 将车辆底边投影到 BEV
  -> 根据车辆与车道区间的重叠判断所属车道
  -> 绘制最终结果和 debug 图
```

### 1. 图像读取与基础通道准备

程序首先从 `config.py` 中读取固定输入路径 `image/cim019.jpg`。图像读入后，会生成灰度图、模糊图和 HSV 图像。

灰度图主要服务于边缘检测和 Sobel 纹理计算；HSV 图像则用于车道线和车辆的颜色分割。相比直接在 BGR 空间中阈值分割，HSV 更适合描述“白色低饱和高亮”“黄色高饱和特定色相”“暗色车辆低亮度”等视觉特征。

### 2. 车道线颜色提取

车道线检测从 HSV 颜色 mask 开始。当前逻辑将车道线分为两类：

- 白色车道线：低饱和度、高亮度。
- 黄色车道线：Hue 位于黄色范围，同时具有较高饱和度和亮度。

两类 mask 合并后得到 `lane_color_mask`。这个 mask 后续既用于鸟瞰图中的车道边界直方图，也用于车辆检测阶段剔除车道线干扰。

### 3. 消失点估计与 IPM 构建

为了把透视图转成更容易分析的鸟瞰图，程序先使用 Canny 提取边缘，再用 `cv2.HoughLinesP` 检测候选线段。

候选线段中会过滤掉太短或接近水平的线段，因为高速公路透视场景中的车道线通常会向消失点汇聚，不应接近水平。随后使用 RANSAC 随机抽取两条线段，计算它们延长线的交点作为候选消失点，并统计有多少线段“近似经过”这个点。支持线段最多的交点会被选为最终消失点。

如果消失点估计失败，程序会回退到默认 IPM 梯形，保证流程可以继续运行。消失点成功时，IPM 源梯形会根据消失点和图像底部左右边界动态构建。这样可以让鸟瞰图尽量覆盖主路面区域。

最终通过 `cv2.getPerspectiveTransform` 得到透视矩阵 `H_mat`，再用 `cv2.warpPerspective` 生成 BEV 鸟瞰图。

### 4. BEV 车道边界检测

在鸟瞰图中，车道线的透视收敛被显著减弱，车道边界更接近竖直线。程序将 `lane_color_mask` 投影到 BEV 后，先沿竖直方向膨胀，把断续的虚线段连接成更连续的竖向结构。

随后对 BEV mask 做列方向统计：每一列中非零像素越多，说明这一列越可能对应车道线。程序对列直方图进行平滑，然后用一维寻峰算法找出稳定的车道线峰值。

内侧白色虚线通常可以通过 BEV 直方图稳定检测，但外侧黄色隔离线和右侧路肩白线在 IPM 梯形边缘附近信号可能变弱。因此项目额外在原图坐标中检测两条外侧实线：

- 黄色隔离线：在黄色 mask 中取最大黄色轮廓，并用 `cv2.fitLine` 拟合整条线。
- 右侧路肩线：在白色 mask 中筛选右侧、竖向跨度足够、延伸到下半幅且方向指向消失点的轮廓。

最后，程序将内侧 BEV 峰值和外侧实线投影位置合并，按 BEV 横向坐标排序，得到完整车道边界。当前图像中最终形成 6 条边界，对应 5 个车道。

### 5. 车辆分割设计

车辆检测没有使用深度学习模型，而是采用传统图像处理方式。核心思想是：先限定车辆可能出现的主路 ROI，再使用多种 HSV 和纹理分支覆盖不同类型车辆。

车辆 ROI 聚焦右侧主道区域，避开中央隔离带和右侧高墙。由于中央黄色隔离带下半段为黄色，上半段可能是灰色混凝土，程序会先基于黄色 mask 找到隔离带种子，再沿其主方向延伸，构造一条较宽的 `median_band`，并从车辆 ROI 中扣除。

车辆分割主要由以下分支组成：

- 暗色车分支：车辆亮度显著低于路面时，通过 `V` 通道低亮度阈值提取。
- 彩色车分支：饱和度较高的车辆通过 `S` 通道提取，同时剔除黄线和白线区域。
- 亮色车分支：白色、银色车辆通常高亮低饱和，因此使用高 `V`、低 `S` 条件提取，并通过车道线走廊排除白色车道线。
- 远处灰白小车分支：顶部远车像素少、亮度不一定达到亮色车阈值，因此在远处 ROI 内放宽亮度条件，并要求局部明暗对比和边缘密度。

这些分支并不是直接输出车辆，而是先合并成 `vehicle_mask`。为了减少路面阴影和平滑区域误检，暗色、彩色、亮色和远处灰白车辆都要经过 Sobel 边缘密度约束。车辆往往具有车窗、车灯、轮廓等丰富纹理，而单纯路面阴影通常纹理较弱。

### 6. 形态学、凸包与连通域过滤

车辆 mask 合并后，程序先做一次开运算去除小噪声，再做闭运算连接车身内部碎片。之后对轮廓进行选择性凸包填充：

- 大轮廓保持原形，避免过度扩张。
- 小/中轮廓使用凸包填充，使被颜色或纹理切碎的车身重新合并。

接下来使用连通域统计得到候选 bbox，并通过多个条件过滤：

- 面积随纵向位置变化：远处车辆允许面积较小，近处车辆允许面积较大。
- 长宽比限制：过滤过扁或过窄的非车辆结构。
- bbox 尺寸上限：避免大面积路面或墙体被误检。
- 填充率限制：过滤过于稀疏的候选区域。
- 黄色比例限制：排除黄色护栏或隔离带残留。

对于贴近画面顶部的远处车辆，程序额外放宽了小目标的长宽比上限，因为这类车辆可能被图像上边缘截断，只剩较扁的车尾或车顶区域。

### 7. 车辆所属车道判断

早期版本只使用车辆 bbox 的底部中心点判断所属车道：将底部中心点投影到 BEV，看它落在哪两个车道边界之间。这种方式简单，但遇到车辆压线、变道、跨车道或投影误差时不够稳定。

当前版本使用车辆底边范围进行判断。对每个车辆 bbox，程序取底边左点、底边右点和底部中心点，并投影到 BEV。底边左右点形成车辆在 BEV 中的横向占用范围。

随后程序计算这个车辆横向范围与每个车道区间的重叠长度：

```text
vehicle_range = [vehicle_left_x, vehicle_right_x]
lane_range    = [lane_left_x, lane_right_x]
overlap       = intersection(vehicle_range, lane_range)
```

车辆会被归入重叠长度最大的车道。如果多个车道重叠长度相同，则使用车辆底部中心点与车道中心的距离作为 tie-breaker。如果车辆范围与所有车道都没有重叠，则归入中心点最近的车道作为兜底。

这样可以保证每辆检测出的车最终只归属一个车道，同时让跨线车辆更倾向于被分配到车身占比更大的车道。当前结果中检测到 11 辆车，5 个车道计数总和也为 11。

### 8. 可视化输出

最终结果图 `result.jpg` 会绘制：

- 黄色线：所有检测到的车道边界，并延伸到画面顶部和底部。
- 红色粗线：车辆数最多的车道两侧边界。
- 绿色框：检测到的车辆 bbox。
- 左上角文字：总车辆数、车道数和每个车道车辆数。

同时，程序会输出多张 debug 图，用于观察每个关键阶段是否正常工作。

## 待解决问题

- 当前算法主要针对固定样例图像调参，泛化到不同摄像头角度、不同道路结构、夜间或雨雪场景时可能需要重新调整阈值。
- 车辆检测基于传统图像处理，遇到严重遮挡、车辆颜色与路面过近、阴影强烈或远处车辆极小的情况时，仍可能出现漏检或误检。
- 车道归属目前保证每辆车只计入一个车道，但对于真实变道车辆，未来可以增加“跨车道车辆”标记。
- 当前输入路径固定为单张图片，后续可以扩展为命令行参数、批量图片或视频流处理。
